# MONOLITH OS v1500 - ГИБРИДНАЯ СИСТЕМА (ANDROID NATIVE + PYTHON WEB)

Мощная гибридная оффлайн-платформа детекции движения, трекинга и сопоставления сигнатур (компьютерного зрения) на базе алгоритма ORB и матричной гомографии.

Проект включает в себя две синхронизированные версии:
- **Версия А**: Высокопроизводительное нативное Android-приложение (Kotlin, Jetpack Compose, CameraX) с возможностью запуска локального фонового сервера управления (порт 5002) в качестве беспроводного пульта.
- **Версия Б**: Веб-интерфейс (PWA), работающий локально в Termux (Android 10+) с захватом потока через WebRTC и оффлайн кешированием.

---

## 🏗️ СТРУКТУРА ПРОЕКТА
- `vision.py` — Ключевой CV-модуль (ORB, BFMatcher, детекция движения, зоны исключения). Единый для обеих версий.
- `server.py` — Flask-сервер связи, аналитики и телеметрии (порт 5002).
- `static/` — Статические ресурсы адаптивного PWA-интерфейса (HTML, CSS вьюпорты, манифест, service-worker).
- `setup_monolith.sh` — Скрипт автоматического развертывания среды Termux одной кнопкой.

## 📦 1. УСТАНОВКА И ОПТИМИЗАЦИЯ ВЕБ-ВЕРСИИ (TERMUX)

Для быстрой автоматической установки зависимостей (Python, numpy, OpenCV, Flask) и запуска сервера откройте Termux и выполните команду:

```bash
curl -sSL https://raw.githubusercontent.com/UFOdom/monolith/main/setup_monolith.sh | bash
```

### Преимущества PWA (Kiosk Mode)
После первого открытия веб-интерфейса в браузере (например, Chrome):
1. Откройте контекстное меню браузера (три точки).
2. Выберите пункт **«Добавить на главный экран»** (или «Установить приложение»).
3. На рабочем столе появится иконка. При ее запуске приложение откроется во весь экран (без адресных строк, вкладок и панелей), имитируя нативный APK-клиент.

### Настройка Виджета (Termux:Widget)
Скрипт установки автоматически конфигурирует ярлык во встроенном плагине `Termux:Widget`.
- Для быстрого старта просто вынесите виджет Termux на рабочий стол вашего телефона и выберите `monolith.sh` — сервер и интерфейс запустятся автоматически.

## 📱 2. НАТИВНОЕ ANDROID-ПРИЛОЖЕНИЕ (APK - ВЕРСИЯ А)

Нативная версия построена на базе современного реактивного стека Jetpack Compose:
- **Встроенная камера CameraX**: Прямой доступ к аппаратному ускорению устройства без WebRTC задержек.
- **Конфигурируемые зоны**: Сенсорная отрисовка зон детекции движения пальцами прямо на видеосенсоре.
- **Интегрированный Web-сервер**: Встроенный сервер (порт 5002) в Android-приложении позволяет управлять датчиком детектора удаленно с ПК или другого устройства из одной Wi-Fi сети.

## 🛠️ ОГРАНИЧЕНИЯ И РЕШЕНИЕ ПРОБЛЕМ (TROUBLESHOOTING)

1. **Ограничение производительности Termux**: Так как Termux не имеет прямого нативного доступа к аппаратному GPU-ускорению системной камеры, обработка Веб-версии ведется через софт-захват `getUserMedia` 15 FPS. Для стабильности разрешение анализа снижено до 480x360 (качество отрисовки bounding box сохраняет 1080p).
2. **Фоновый спящий режим**: На оболочках MIUI, HyperOS, OneUI система Android может принудительно "убивать" процесс софта Python внутри Termux. Чтобы решить это:
   - Откройте Настройки телефона -> Приложения -> Termux -> Контроль активности -> **Безограничений**.
3. **OpenCL**: Если ваше ARM-устройство поддерживает аппаратное OpenCL ускорение графики, Python-сервер автоматически перенаправит часть расчетов ORB-сигнатур на видеочип.
